HTML Editing

  • Current spec TS_60_HTMLeditor
  • This spec is still valid and describes "plain old" CSS editor and should be tested as well

"Life" editing

  • Task: CSS Styles window population
    1. Open some html file in editor and run it
    2. When the page is running, place cursor somewhere in the editor
    • Result: CSS Styles window is populated and contains styles for element based on cursor position. Try to move cursor and make sure that CSS Styles window and Rule editor change their content according to cursor position
  • Task: Go To Sources from Navigator
    1. Invoke right click menu on some "green" element in Navigator
    2. Select Go To Sources
    • Result: Cursor is put inside the given file on a correct position
  • Task: Go To Sources from Navigator II
    1. Invoke right click menu on some "dynamic" element in Navigator
    • Result: Go To Source is disabled
  • Task: Properties window
    1. Put cursor inside some element and switch to Properties window (Window|Properties)
    2. The window contains list of HTML attributes
    3. Existing HTML attributes are shown together with their values
    • Result: Select some attribute and modify it. Make sure that the changes are applied to the HTML file

Import styles and Go To Declaration

Create a new HTML5 project and use following snippets
HTML File: 
<!DOCTYPE html>
<html>
    <head>
        <title></title>
        <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    </head>
    <body>
        <div class="test">TODO write content</div>
    </body>
</html>

Newcss file: 
.root { 
    display: block;
}

.test{
    content:"c";
}

Newcss1 file: 
.test{
   color:red;
}
  • Task: Add Reference to CSS Style
    1. Open the html file (in HTML5 project) in NetBeans
    2. Click on the bulb icon, list of actions is offered.
    3. Click on "Add reference to ... newcss.css"
    • Result: New LINK element is added to the HTML source with href attribute pointing to the css file

  • Task: Go To Declaration
    1. Ctrl+Click on the class attribute
    2. List of found "test" class rules is displayed together with line number. If some file is not referenced from the HTML file, message "unrelated" is added.
    3. Click on the first item
    • Result: Given css file is opened in editor and cursor is added to the appropriate possition. Repeat the same for ID attribute

  • Task: New Rule
    1. Change the div's class attribute to "test3"
    2. Click on the bulb icon, it offers you to create new rule (for both css files).
    3. Click on the 2nd one (the one not linked in html)
    • Result: New LINK element is added to the HTML source with href attribute pointing to the css file and the css file contains the new rule
  • Task: New class in new file
    1. Create a new HTML/JavaScript project, all default values
    2. Add some class attribute to the div element
    3. Click on the bulb icon, it offers you to create new rule and new css file at the same time.
    4. Confirm this action
    • Result: New class rule is created in a new css file and the file is linked from the html file
  • Task: New class in existing file
    1. Use the source from case above
    2. Change the value of class attribute
    3. Click on the bulb icon, it offers you to create new rule in all linked css files
    4. Confirm this action for some linked css file
    • Result: New class rule is created in the css

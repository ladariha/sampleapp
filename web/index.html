Easel work flow

This section contains steps and links to specifications for expected work with Easel. You should try it with both, Chrome and Embedded WebKit browser. Note that some terms or screens may differs from the build you are testing.

For testing purposes, use (link)

Getting ready

  • test Chrome extension, follow these instructions
  • Get to know Embedded browser, read and try these instructions
  • try basic operations with client side project, follow these instructions

Run

  • Run the page from IDE. Default configuration should be set to Embedded WebKit browser. This browser opens below your editor area in NetBeans. If you choose to run it in Chrome, new tab is opened and you can see NetBeans icon in Omnibar and message that NetBeans plugin is debugging this tab.
  • When the page is running, you can see that Navigator window has colored icons, CSS Styles and Rule Editor are opened
  • If not already, open Output window which should contain 2 tabs: WebKit Protocol and Browser log which prints logging from JavaScript (either triggered via console object or errors and stacktraces)
  • Navigator DOM features, follow these instructions

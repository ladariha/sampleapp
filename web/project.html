HTML/JavaScript project related

New Project

  • Task: Create empty project
    1. Invoke New Project dialog
    2. select HTML/JavaScript and HTML5 Application
    3. Finish with default values
    • Result: New project is created. It contains single index.html file and 4 folders: config, nbproject, public_html and test. Make sure that default settings correspond to the image above
  • Task: Create project with standard template
    1. Invoke New Project dialog
    2. select HTML/JavaScript and HTML5 Application
    3. In the 3rd step Site Template, select Download Online Template
    4. Select HTML5 Boilerplate and check template's description below
    5. Press Next
    6. In the JavaScript Libraries step, there are already selected some libraries that are bundled with the template. Check that you cannot remove them
    7. Use text field next to Available to filter list of libraries
    8. Select e.g. Dojo and click to the Version column. List of available versions of Dojo is offered, pick some and add it to the project
    9. Change Libraries Folder to "js/libs_custom" and finish the process
    • Result: New project is created. Make sure it contains js/libs and js/libs_custom (withi Dojo library) folders as well as some default files from HTML5 Boilerplate (icons, readme.md....)
  • Task: Create project with custom template
    1. Having the project from previous step, right click on it in Projects window
    2. Select Save as Template
    3. In the opened dialog, you can specify what should be in the template. Uncheck the same items as on the image below
    4. Select some template name and location
    5. Press Finish
    6. Now invoke New Project dialog, sleect HTML/JavaScript and HTML5 Application
    7. Select "Select Archive" and point to the location of your template, press Next
    8. In the JavaScript Libraries step, there are already selected some libraries that are bundled with the template. Check that you cannot remove them
    9. press Finish
    • Result: New project is created from custom template. Make sure it doesn't contain the files/folders unchecked in Save as template dialogs

New Project with sources

  • Task: Create Project with existing sources
    1. (have some sources ready in folderA)
    2. Invoke New Project dialog and select HTML/JavaScript and HTML5 Application with existing sources
    3. Set Site Root to folderA
    4. Project name is set to "folderA"
    5. Default project directory is set to "folderA"
    6. Finish the dialog
    • Result: New project is created. Site Root is set to "folderA" , Unit Tests to "test". Web Root is set to "/folderA". Right after project creation, you should resolve project's problem.
  • Task: Resolve project problem I - III
  • Task: Create Project with existing sources II
    1. (have some sources ready in path/folderA)
    2. Invoke New Project dialog and select HTML/JavaScript and HTML5 Application with existing sources
    3. Set Site Root to path/folderA
    4. Project name is set to "folderA"
    5. Default project directory set to some other directory, e.g. "path/projectName
    6. Finish the dialog
    • Result: New project is created. Site Root is set to "folderA" (relative path), Unit Tests to "test". Web Root is set to "/projectName".
  • Task: Import AngularJS project
    1. Extract archive from data/angular_struct.zip to some folder "folderA"
    2. In NetBeans, invoke New Project dialog and select HTML/JavaScript and HTML5 Application with existing sources
    3. Set Site Root to folderA/app
    4. Project name is set to "AngularJsApplication"
    5. Finish the dialog
    • Result: New project is created. Site Root is set to "app" , Unit Tests to "test" and Configuration Files to "config"

Project settings

You may use this applications for testing following cases. Just remember to revert them back :)
  • Task: Site Root and Unit Tests folders
    1. Go to Project Properties (right click on project) and select Sources category
    2. Modify Site Root
    3. Confirm changes
    • Result: List of files under Site Root in Projects window is changed and contains only files from the selected Site Root. Repeat the same with Unit Tests folder. Node that this node is not displayed in Projects window if it doesn't contain any file
  • Task: Main file to run
    1. Go to Project Properties (right click on project)
    2. Change Start file
    3. Right click on project and select Run
    • Result: Start file is opened in browser
  • Task: Main file to run II
    1. Go to Project Properties (right click on project)
    2. Change Start file to some non-existing file
    3. Right click on project and select Run
    • Result: Warning message saying that main file to run cannot be found is opened. If you confirm the dialog, Project Properties dialog is opened and you can change it back
  • Task: Server settings
    • Result: (not yet implemented)
  • Task: Web root settings
    1. Go to Project Properties (right click on project) and select Run category
    2. Change Web Root to e.g. "myweb"
    3. Right click on project and select Run
    • Result: The project is running and URL contains e.g. localhost:8383/myweb/[startfile]
  • Task: Remote files
    1. Open this project in NetBeans
    2. In Projects window, there is a node called Remote Files, make sure it contains 3 files
    3. Modify some html file and add a new <script> tag with src attribute pointing to some URL address
    4. Save the file
    • Result: Remote Files contains the new script. Try to open it in IDE, it should be read-only (see issue 217664)
  • Task: Browser settings
    1. Go to Project Properties (right click on project) and select Run category
    2. By default (if you run in browser with NetBeans integration), Auto Refresh and Highlight Selection are enabled
    3. Change browser to some without NetBeans integration
    • Result: The 2 options are no longer available
  • Task: Auto Refresh
    1. Go to Project Properties (right click on project) and select Run category
    2. Uncheck the Auto Refresh option
    3. Run some html file
    4. File is opened in browser, now edit it in IDE and save
    • Result: File is not refreshed in browser
  • Task: Import JavaScript files
    1. open Project Properties for some HTML5/JavaScript project
    2. Select JavaScript files category
    • Result: There is a list of available JS libraries, make sure it all works the same as in New Project wizard. In the right column there is a list of already used JS files in the project and these files cannot be removed/altered from this dialog

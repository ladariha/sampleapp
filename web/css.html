Life Editing - CSS Styles & Rule Editor

CSS Styles window requires html to run in browser.
  • Task: Selection styles
    1. Run this file in browser (test both, Chrome & JavaFX)
    2. Turn Selection Mode on
    3. Select the title "NetBeans Testing App"
    • Result: CSS Styles window shows in Property Summary list of applied properties to the title. Note (and make sure) that if you select some element in Navigator window, CSS Styles Window and Properties window are populated as well
  • Task: Property Summary interaction
    1. Having selected heading from the previous step, check that border is expandable
    2. If you click on some property in Property Summary, Rule Editor is populated and shows css rule where the selected property is defined (try "font-size" and "color" and make sure Rule Editor shows different rules)
    3. If you double click on some item, the appropriate file with the CSS property definition is opened in editor
    4. Invoke right click context menu on some item in CSS Styles and select Go To Source.
    • Result: The appropriate file with the CSS property definition is opened in editor
  • Task: Styles section
    1. Having selected heading from the previous step, check that Styles shows list of applied css rules for given element
    2. If you click on some rule in Styles, Rule Editor is populated and shows the selected css rule
    3. If you double click on some item, the appropriate file with the CSS property definition is opened in editor
    4. Invoke right click context menu on some item in Styles and select Go To Source.
    • Result: The appropriate file with the CSS property definition is opened in editor
  • Task: Overridden Properties
    1. Have some HTML element which styles are overridden (e.g. use css selector for element and for class and add "color" property to both of them with different value)
    2. Run page and select the element in Navigator
    3. Select the 2 styles in CSS Styles window
    • Result: One property is crossed (see image below)


  • Task: Indication of state of CSS properties
    1. Run this file
    2. Select some li item in Navigator and click on "body" na Styles
    • Result: Applied css properties are black, overriden are black crossed, unknown by browser (or erroneous) are red and unused are gray crossed
  • Task: Editing selected item
    1. Having selected heading from the previous step, check that Styles shows list of applied css rules for given element
    2. If you click on some rule in Styles, Rule Editor is populated and shows the selected css rule
    3. Modify some property and make sure it changes the correct css file on the correct line. If you modify css style, the selected element should be instantly updated in browser and in CSS Styles window
    • Result: Everything works
  • Task: CSS Styles: Document View
    1. In CSS Styles window, switch to Document
    2. There is a list of all applied css files
    3. Each file can be expanded to show all rules defined inside it
    4. Try double click and context menu action to go to source as above
    5. Type some search parameter
    • Result: Everything works. If you specify some search parameter, list is reduced to matching items
  • Task: Rule Editor and CSS Styles interaction
    1. Select some property in Property Summary in Selection view
    2. Rule Editor should display all properties from the rule where the selected property is defined
    3. Edit some property in the Rule Editor.
    • Result: Everything works. Properties can be modified directly from the Rule Editor. It should work the same if you select some DOM element (right column) or rule (left column) from Style Cascade section. If you switch to Document View and select some rule, Rule Editor lists all properties from the given rule
  • Task: New Control toolbar in Rule Editor
    1. Select some element either from Navigator window or using Select Mode
    2. Select some of its styles from CSS Styles
    3. Rule Editor is populated. You can see action icon in the top right corner with following actions: Add Property, Create Rule, Delete Rule
    • Result: Try each of this action and make sure the you can add property to the rule, delete selected rule or add a new rule to the file. Field "Stylesheet" has to correspond with stylesheet where the selected rule is defined. "At Rule" lists @media queries in css file and allows to insert a rule to the specific @media query
  • Task: Simple list of values in Add Property and Rule Editor
    1. Select some item in Navigator and some of its styles in CSS Styles
    2. Rule editor is populated with given rule
    • Result: Test that for some know properties there is a list of possible values (color, float etc.) and also there is "none". If you select none then the property is removed from the rule
  • Task: Cursor position in editor is in sync with CSS Styles window
    1. Run html file
    2. Place cursor into some element in the source
    • Result: CSS Styles window contains styles for the given element
  • Task: Rule Editor - Add Property
    1. Open /stylesheets/base.css in this project
    2. Place cursor inside some rule. Rule Editor window should open and show all properties in the rule
    3. Press the "+" next to the "Add Property" button
    4. New dialog is opened, try to change change the way properties are order using the "Show Property Categories" switch button
    5. Already used properties can't be modified in this dialog
    6. Select some line and type in to the value cell
    7. Try to filter out properties using the filter text field
